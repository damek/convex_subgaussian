2.1 Reader-facing theorem surface

The public Lean API is built around the random-variable formulation of the comparison problem. The endpoint file exposes the admissible Gaussian scales and their infimum.

Definition 1 Endpoint quantities
✓
#

Lean declarations

  • cStar

The declaration AdmissibleScale says that a positive number \(c\) is admissible if every centered real random variable \(X\) with the two-sided tail bound

\[ \mathbb {P}(|X| {\gt} t) \le 2 e^{-t^2/2} \qquad (t \ge 0) \]

is convex-dominated by the centered Gaussian law of standard deviation \(c\). The quantity \(c_\star \) is then defined as the infimum of all such admissible scales.

Definition 2 Explicit Gaussian comparison constants
✓
#

Lean declarations

  • gaussianScale

The explicit constant is built from the sub-Gaussian tail cap

\[ s_G(t) := \min \{ 1, 2e^{-t^2/2}\} , \qquad t_0 := \sqrt{2 \log 2}, \]

the total tail mass

\[ A := \int _0^\infty s_G(t)\, dt, \qquad B := A/2, \]

the unique threshold \(a {\gt} t_0\) solving

\[ H(a) = a s_G(a) + \int _a^\infty s_G(t)\, dt = B, \]

the height \(p_0 := s_G(a)\), the Gaussian tail quantile \(z\) satisfying \(\overline{\Phi }(z) = p_0\), and finally

\[ c_0 := \frac{B}{\varphi (z)}. \]

The public support file exposes these definitions exactly so the reader can see which constant is being proved optimal.
